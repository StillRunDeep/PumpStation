# AG0-0 用户对话 Agent

> 功能分类：**用户交互 + 数值计算**
> 依据：香港渠务署《雨水排水手册（第五版）》第4章、第7章
> 代码文件：`src/agents/ag00-validate.js`

---

## 一、职责概述

AG0-0 是雨水泵站智能设计系统的**入口 Agent**，负责：
1. 通过界面输入收集设计参数
2. **执行暴雨分析计算（步骤1）**：根据重现期和暴雨历时计算降雨强度
3. **执行径流估算计算（步骤2）**：根据汇水区特性计算设计流量 Q（或直接接收总排水量）
4. 参数合法性校验
5. 将计算结果传递给下游 Agent

---

## 二、在系统中的位置

```
用户输入 → AG0-0 → 暴雨分析 → 径流估算 → Q
```

## 四、计算逻辑

### 模式一：直接输入总排水量（适用于案例复现）

当用户直接输入 `Q_total`（水泵最高总排水量，m³/s）时，跳过暴雨分析和径流估算步骤。

**单泵流量计算**：
```
Q_pump = Q_total / N    （m³/s）
```

---

### 模式二：暴雨分析→径流估算（适用于新设计）

当 `Q_total` 未指定时，执行以下步骤：

#### 步骤 1：暴雨分析（IDF 公式）

**依据**：手册第4.3节

根据用户选择的分区和重现期，从常数表（见 00-计算定义总纲 §6.1）获取 IDF 常数 a, b, c：

```
i = a / (t_d + b)^c    （mm/h）
```

**气候变化修正（可选）**：

若用户指定 Δi（气候变化降雨增加量），则设计暴雨深度需修正：

```
P = i × t_d × (1/60) × ARF × (1 + Δi/100)    （mm）
```

其中：
- Δi = 0   （当前气候）
- Δi = 10  （21世纪中叶RCP4.5）
- Δi = 20  （21世纪末RCP4.5）
- Δi = 25  （21世纪末RCP8.5）

**IDF常数表（手册表3a-3d）**：

| 分区 | 重现期 | a | b | c |
|------|--------|---|---|---|
| 香港天文台总部 (tungkl) | 10年 | 14.610 | 13.055 | 0.741 |
| 香港天文台总部 (tungkl) | 50年 | 17.903 | 14.896 | 0.732 |
| 香港天文台总部 (tungkl) | 200年 | 20.558 | 16.150 | 0.725 |
| 大帽山 (tai-mo-shan) | 10年 | 16.610 | 13.055 | 0.741 |
| 大帽山 (tai-mo-shan) | 50年 | 19.903 | 14.896 | 0.732 |
| 大帽山 (tai-mo-shan) | 200年 | 22.558 | 16.150 | 0.725 |

**示例**：大帽山分区，T=10年，t_d=120min
- 查表：a=16.610, b=13.055, c=0.741
- i = 16.610 / (120 + 13.055)^0.741 ≈ 2.08 mm/h

---

#### 步骤 2：径流估算（推理法）

**依据**：手册第7.5.2节

```
Q_p = 0.278 × C × i × A × ARF    （m³/s）
```

**转换为 m³/h 用于后续计算**：
```
Q = Q_p × 3600    （m³/h）
```

**面积折减系数**：
```
ARF = 1.00                        （A ≤ 25 km²）
ARF = 1.547 / (A + 280.11)       （A > 25 km²）
```

**示例**：A=1.0km², C=0.90, i=2.08mm/h, ARF=1.0
- Q_p = 0.278 × 0.90 × 2.08 × 1.0 × 1.0 = 0.52 m³/s
- Q = 0.52 × 3600 = 1872 m³/h

---

#### 步骤 2b：集流时间计算（布兰兹贝-威廉公式）

**依据**：手册第7.5节（天然集水区）

```
t_c = 0.14465 × L / (H^0.2 × A^0.1)    （分钟）
```

其中：
- t_c = 集流时间（分钟）
- L = 沿天然流向的水平距离（m）
- H = 平均坡降（m/100m）
- A = 集水区面积（km²）

**注意**：本参数主要用于天然集水区。对于城市排水系统，径流估算可跳过集流时间计算，直接使用推理法峰值流量公式。

---

### 步骤 3：单泵流量计算

```
Q_single = Q / N    （m³/h）
```

## 五、参数校验规则

### AG0-0 参数范围定义

| 参数 | 最小值 | 最大值 | 单位 | 依据 | 错误提示 |
|------|--------|--------|------|------|----------|
| Q_total | 0.1 | 100 | m³/s | 用户输入 | 总排水量应在 0.1-100 m³/s 范围内 |
| Z | 4 | 12 | 次/小时 | 手册第14.6.1节 | 每小时启动次数应在 4-12 次范围内 |
| N | 1 | 6 | 台 | 手册第14.6.2节 | 工作泵台数 N 应为 1-6 之间的整数 |
| N_spare | 0 | 3 | 台 | 手册第14.6.2节 | 备用泵台数应在 0-3 范围内 |
| V_design | 100 | 500000 | m³ | 用户输入 | 设计水缸容量应在 100-500000 m³ 范围内 |
| Z_bottom | -50 | 10 | mPD | - | 池底标高应在 -50-10 mPD 范围内 |
| Z_sump | -50 | 10 | mPD | 手册第14.6.3节 | 集水坑底标高应低于池底标高 |
| D | 5 | 30 | m | 用户输入 | 设计水缸深度应在 5-30 m 范围内 |
| Z_discharge | -10 | 50 | mPD | - | 排放口标高应在 -10-50 mPD 范围内 |
| Δi | 0 | 30 | % | 手册第6.8节 | 气候变化降雨增加量应在 0-30% 范围内 |
| H | 0.1 | 100 | m/100m | 用于集流时间计算 | 平均坡降应大于 0 |
| L | 10 | 10000 | m | 用于集流时间计算 | 最长流径水平距离应在合理范围内 |

### 水位关系校验

```
Z_discharge = Z_bottom + D  （排放口标高等于池底+深度）
Z_bottom < Z_discharge       （池底应低于排放口标高）
Z_sump < Z_bottom            （集水坑底应低于池底）
```

### 警告条件

| 条件 | 警告信息 |
|------|----------|
| N > 6 | 工作泵台数超过 6 台，效率可能下降 |
| Q_pump > 3 m³/s | 单泵流量偏大，请确认输入参数 |

```json
{
  "Q_total": "<水泵最高总排水量，m³/s>",
  "Q_pump": "<单泵设计流量，m³/s>",
  "N": "<工作水泵台数，台>",
  "N_spare": "<备用泵台数，台>",
  "Z": "<每小时允许启动次数，次/小时>",
  "V_design": "<设计水缸容量，m³>",
  "Z_bottom": "<池底标高，mPD>",
  "Z_discharge": "<排放口标高，mPD>",
  "D": "<设计水缸深度，m>",
  "Z_top": "<池顶标高（计算值），mPD>",
  "mode": "<输入模式：direct/rainfall>"
}
```

下游 Agent 接收方：
- **AG1-1**：接收 V_design, Z_bottom, D, N, Z, Q_pump → 调蓄池计算
- **AG1-2**：接收 Q_pump, Z_stop, Z_discharge → 水泵选型
- **AG1-2**：接收 N → 维护间尺寸
