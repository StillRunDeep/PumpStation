# 🚀 AG3-1 模块：2D 真实比例平剖面预览引擎 PRD

## 1. 核心架构与全局坐标系 (XYZ 规约)
彻底抛弃抽象的像素坐标系，必须建立基于物理真实尺寸的 `SCALE`（像素/米）映射。

* **分层界限 (Z-top 逻辑)**：
    * 建筑空间不再以 Z=0 划分，而是以 **`Z-top` (水池顶板/机房底板标高)** 为绝对分界线。
    * **`Z > Z-top`**：上方为维护间（Pump Room）。布置阀门、总管及控制设备。
    * **`z-bottom < Z < Z-top`**：为水池。标注pool模块的水位结果。
    * **`z-sump < z < z-bottom`**:是集水坑（Wet Well）。布置潜污泵。
* **平面图 (XY 坐标系)**：
    * `X 轴 (水平)`：对应物理宽度 **W**。水泵沿 X 轴并排等距排布。
    * `Y 轴 (垂直)`：对应物理长度 **L**。支管从水泵引出后，沿 Y 轴向下平行延伸。
* **剖面图 (Z 坐标系)**：
    * `Z 轴 (竖向)`：对应物理标高 (Elevation)。反映 `Z-top`、池深、水位及机房净高。

## 2. 前置数据输入 (Data Payload)
渲染引擎仅接收合并后的 JSON 进行渲染水合：
1. **空间边界**：维护间 `(L, W, h_room)`，集水池 `(L_pool, D_pool)`。
2. **绝对标高**：`Z-top` (顶板标高)、坑底标高、停泵/启泵/报警等水位标高。
3. **实体尺寸表**：水泵长宽 `(w, d)` 及接口 DN；阀门及管件的实体端到端长度。
4. **无向拓扑链**：如 `[ { node1: 'pump1', node2: 'cv1' } ... ]`。

## 2.5 空间层规范（房间关系 Room Layer）

### 默认空间关系：上下叠加

```
┌──────────────────────────────┐  Z > Z-top
│     泵房维护间 (pump_room)    │  布置阀门、总管、机电设备
│                              │
│  ○   ○   ○  ← 穿楼板孔位    │  孔位在集水坑投影范围内
└──────┬───┬───┬───────────────┘  Z = Z-top （楼板/顶板）
       │   │   │   ← 管道穿楼板向下
  ┌────┴───┴───┴────────────┐
  │    集水坑 (wet_well)     │  Z-sump < Z < Z-top
  │  [P1]   [P2]   [P3]    │  水泵布置于此
  └─────────────────────────┘
┌──────────────────────────────┐
│      水池 (water_pool)       │  集水坑的外围蓄水空间
└──────────────────────────────┘
```

### 关系约束

| 约束 | 规则 |
|------|------|
| **投影包含** | 集水坑的平面范围（XY）必须完全在泵房维护间的平面投影内 |
| **穿楼板对齐** | 每台泵的穿楼板孔位 X 坐标 = 该泵在集水坑内的中心 X |
| **楼板分界** | Z-top 是唯一分界面；维护间在上，集水坑/水池在下 |
| **管道连通** | 所有管道从维护间穿过楼板（Z-top）向下接泵；**穿楼板点是弯头节点**（`elbow: top → in`） |

### 房间层驱动的尺寸规则

泵房维护间的 **L 方向**（沿泵排方向）由以下规则共同确定，取最大值：

```
L = max(
  L_pipe,          // 管道及阀件占位（AG2-1 计算）
  L_pumps,         // 泵排占位（泵宽 + 间距）
  L_sump_margin    // 集水坑占位 + 两侧管道余量（本规则）
)
```

**`L_sump_margin` 规则**（穿楼板管道驱动）：

```
// 每侧余量 = 最外侧穿楼板管道外壁到维护间侧墙净距 ≥ pipeToWall
// 因此：侧墙距最外侧管道中心 = DN_branch/2 + pipeToWall
e_pipe = DN_branch/2 + pipeToWall_min

L_sump_margin = sump_width              // 集水坑净宽（含隔墙）
              + 2 × e_pipe              // 两侧管道余量
```

泵房维护间的 **W 方向**（垂直于泵排）由管道链长度决定（AG2-1），不由本规则驱动。

### 可配置关系（扩展点）

默认为"上下叠加"关系（泵房在上、集水坑在下）。  
以下参数可改变空间配置，供未来扩展：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `roomRelation` | `'vertical_stack'` | 上下叠加（当前唯一实现） |
| `sumpAlignment` | `'center'` | 集水坑相对维护间的对齐方式 |
| `pipeDir` | `'down'` | 管道穿楼板方向（默认向下） |

## 3. 双视口平面图规则 (Dual Viewport Plan)
针对集水池与机房比例悬殊的问题，采用双视口策略：

### 3.0 画布总体布局
* AG3-1 画布采用横向三列布局，尽量使用结果卡片的完整宽度：
    * **第一列：总览信息列**，上方放 Minimap，下方放图例。
    * **第二列：聚焦主平面图**，表达维护间、集水坑、泵组和梳齿管线。
    * **第三列：A-A 剖面图**，表达维护间、水池、集水坑和代表支路竖向关系。
* 图例不放在主平面图内部，避免遮挡管线和设备。

### 3.1 鹰眼缩略图 (Minimap) - 全局比例
* **位置与尺寸**：置于画布边缘，设定固定 UI 尺寸（如 150x150px 区域）。
* **渲染内容**：使用独立的微缩比例尺 `SCALE_MINI`，仅绘制外框轮廓，展示“巨大集水池”与“小型维护间”的宏观占位比例，不画内部管道细节。

### 3.2 聚焦主平面图 (Main Plan View) - 梳子式并联
* **取景范围**：采用 `SCALE_MAIN`，最大化聚焦于维护间 `L x W` 范围及接壤的坑口区域。
* **主图边界**：
    * 主平面图只绘制**维护间矩形**和其下方接壤的**集水坑矩形**，用于表达内部设备、支管和总管布置。
    * 主平面图不绘制完整 `L_pool x D_pool` 集水池外框，避免大水池尺度压缩或越界影响设备布置表达。
    * 完整集水池与维护间的真实比例关系只在左上角 Minimap 中表达。
    * 维护间和集水坑边界使用实线表达。
* **集水坑平面尺寸**：
    * 集水坑宽度按泵组外包计算：`W_sump = Σw_pump_i + (N_total - 1) × 0.2m`，其中 `0.2m` 为相邻泵坑之间的隔墙厚度。
    * 集水坑长度按单泵外形长度计算：`L_sump = max(d_pump_i)`。
    * 水泵在集水坑内按 `w_pump + 0.2m 隔墙` 分格布置，穿层孔洞与对应泵中心对齐。
* **组件排布推演 **：
    * **穿层孔洞**：在水泵 X,Y 坐标处，强制绘制圆形虚线（代表管道从 Z-top 下方穿上来）。
    * **梳齿 (支管)**：沿 Y 轴向下，`下一个组件中心 = 当前中心 + (L1+L2)/2 + 直管段预留`。依次绘制止回阀、闸阀。
    * **梳脊 (总管)**：在 Y 轴末端，沿 X 轴横向绘制一根贯通的总管 (`DN_main`)，所有支管在此垂直汇入。汇合点后到出水点的构件也放在总管的最后一个梳齿后面。

    # 布局结构
ELBOW ── TEE ── TEE ── TEE ── TEE ── GV ── CV ── FM ──→ 出水口
  │      │      │      │      │
 CV     CV     CV     CV    CV   ← 阀门（按拓扑顺序）
  │      │      │      │      │
  P      P      P      P    (进水)   ← 齿的末端
[========== 集水坑 ==========]

**拓扑图**：只包含 devices（泵、阀门、汇流点、进水、出水）和 pipes（连接关系）
- **管件推断规则**：
  - 第一个连接点 → **弯头（ELBOW）**，左侧无边
  - 后续连接点 → **三通（TEE）**
- **主管（梳脊）** = ELBOW~TEE 之间的水平线（DN_main）
- **梳齿** = 从各弯头/三通垂直向下：水泵（或进水口）→ 阀门 → 弯头/三通 → 主管
- **TEE 右侧**：GV → CV → FM → 出水口（按拓扑 `valvesAfterJunction` 顺序）

### 3.3 当前实现落地规则
* `pumpsInOrder` 从 topology 中 `roomId='wet_well'` 的 pump 设备推导，并按 `canvasX` 排序；平面泵标签使用拓扑标签（如 `P1`、`P2`、`备`）。
* `buildLayoutMap` 统一输出泵序列、每泵支路链、主管链、旁通链、梳齿 X 坐标和代表支路，供平面图和剖面图共用。
* 每个梳齿的设备顺序由拓扑 BFS 从泵出口追踪至汇流点得到，过滤掉泵本体和汇流点，只绘制中间阀门/仪表。
* 汇流点后的主管设备链由拓扑从汇流点出口追踪至出水节点得到，并压入维护间右侧总管范围内按顺序绘制。
* 平面图尺寸线、比例尺和图例均基于“维护间 + 集水坑”的主图底边，不再引用完整集水池底边 `pool_y2`。

## 3.4 LayoutGraph 规范（拓扑→SVG 的中间层）

### 设计目的

在拓扑（逻辑关系）和 SVG（像素）之间引入一层 **LayoutGraph**：
每个物理构件对应一个 `LayoutNode`，带有坐标、朝向、管件类型和端面位置。
渲染器只负责把 `LayoutNode` 翻译成 SVG 字符串，**不再内嵌任何布局决策**。

```
Topology (nodes / devices / pipes)
        ↓  buildLayoutGraph(topology, geo, inputs)
LayoutGraph (LayoutNode[])
        ↓  renderToSVG(graph, geo)
SVG string
```

---

### 数据结构

```ts
/** 管件/管段的朝向（视图平面内） */
type PlanDir = 'right' | 'left' | 'top' | 'bottom' | 'in' | 'out'

/**
 * 单个物理构件节点
 * 坐标单位：SVG 像素（已乘 SCALE_MAIN）
 */
interface LayoutNode {
  /** 构件类型，决定 SVG 符号 */
  kind: LayoutNodeKind

  /** 构件中心坐标（管件体几何中心，或管道中点） */
  cx: number
  cy: number

  /** 主方向 DN（驱动几何尺寸） */
  dn: number

  /** 次方向 DN（仅三通需要：支管 DN） */
  dnBranch?: number

  /** 弯头：入口方向 */
  fromDir?: PlanDir
  /** 弯头：出口方向 */
  toDir?: PlanDir

  /** 来自拓扑的设备 id（调试/交互用） */
  deviceId?: string

  /**
   * 端面坐标（渲染器计算管道线段时使用）
   * from = 上游端面中心，to = 下游端面中心
   */
  faces?: {
    from: { x: number; y: number }
    to:   { x: number; y: number }
  }
}

type LayoutNodeKind =
  | 'pipe_segment'      // 管道线段（两端面之间）
  | 'elbow'             // 弯头：L形 或 I+圆（穿楼板）
  | 'tee'               // 三通：T形
  | 'gate_valve'        // 闸阀：矩形
  | 'check_valve'       // 止回阀：菱形
  | 'flowmeter'         // 电磁流量计：矩形+圆
  | 'reducer'           // 大小头：梯形
  | 'pump'              // 水泵（集水坑内）
  | 'floor_penetration' // 穿楼板点（等同于 elbow in/out）

/** 完整布局图 */
interface LayoutGraph {
  nodes: LayoutNode[]
  /**
   * 按管路分组，便于按层渲染（先画管道，再画管件，最后画标注）
   * groups: { main: LayoutNode[], branches: LayoutNode[][], return: LayoutNode[] }
   */
  groups: LayoutGroups
}

interface LayoutGroups {
  /** 主管（梳脊）：ELBOW → TEE... → 阀组 → 出水 */
  main:     LayoutNode[]
  /** 支管（梳齿）：每台泵一条，从穿楼板弯头到主管 TEE */
  branches: LayoutNode[][]
  /** 回流管：进水 → 穿楼板 → 弯头 → 主管 ELBOW */
  return:   LayoutNode[]
}
```

---

### 管件类型推断规则（buildLayoutGraph 职责）

| 位置 | 规则 | 产生节点 |
|------|------|---------|
| 主管最左端（第一个连接点） | 拓扑中为"进水/回流"连接，左侧无更多支管 | `elbow`（fromDir='bottom', toDir='right'） |
| 主管中间每个支管连接点 | 拓扑中为泵支路汇入点 | `tee`（支管向下） |
| 支管底端（穿楼板） | 支管进入集水坑，垂直穿过楼板 | `elbow`（fromDir='top', toDir='in'） |
| 主管方向改变 | 拓扑路径中存在转角 | `elbow`（fromDir/toDir 由路径计算） |
| 相邻设备之间 | 两个管件端面之间有物理距离 | `pipe_segment` |
| 支路中阀门 | 来自拓扑 device，type 为已知阀件类型 | 对应 kind |

**约束**：管道线段（`pipe_segment`）**不跨越**任何管件体；
线段的起止 = 前一节点 `faces.to` → 后一节点 `faces.from`。

---

### SVG 渲染映射

```ts
function renderNode(node: LayoutNode, color: string): string {
  switch (node.kind) {
    case 'pipe_segment':    return _l(node.faces.from, node.faces.to, color, sw)
    case 'elbow':           return _elbow(node.cx, node.cy, r, node.fromDir, node.toDir, color, sw, dn_px)
    case 'tee':             return _tee(node.cx, node.cy, dnMain_px, dnBranch_px, color)
    case 'gate_valve':      return _gateValve(node.cx, node.cy, halfLen, horiz, color)
    case 'check_valve':     return _checkValve(node.cx, node.cy, halfLen, horiz, color)
    case 'flowmeter':       return _flowmeter(node.cx, node.cy, halfLen, dn_px, horiz, color)
    case 'reducer':         return _reducer(...)
    case 'pump':            return _rect(...) // 泵矩形
    case 'floor_penetration': return _elbow(node.cx, node.cy, r, 'top', 'in', color, sw, dn_px)
  }
}
```

渲染循环分两遍：
1. **第一遍**：渲染所有 `pipe_segment`（先画管道）
2. **第二遍**：渲染所有管件节点（覆盖在管道之上，保证管件符号完整可见）

### 尺寸来源汇总

| 管件类型 | 关键尺寸 | 来源 |
|---------|---------|------|
| `elbow` | 中心线半径 `r = elbowCTF(DN) * 0.5` | `fitting-dims.js` |
| `tee` | `mhl, mhh = f(DN_main)`；`bw, bl = f(DN_branch)` | `_teeGeom()` |
| `gate_valve` | 面对面长度 `GATE_VALVE_FF[DN]` | `fitting-dims.js` |
| `check_valve` | 面对面长度 `CHECK_VALVE_FF[DN]` | `fitting-dims.js` |
| `flowmeter` | 本体长度 `flowmeterBodyL(DN)` | `fitting-dims.js` |
| `pipe_segment` | 两端面距离（动态计算） | 布局算法 |

### 房间层与机电层分离

| 层 | 职责 | 关键输入 |
|----|------|---------|
| **房间层** | 确定维护间 L×W、集水坑位置和尺寸、穿楼板孔位 | AG2-1 输出 |
| **机电层** | 在房间层确定的坐标系内生成 LayoutGraph | Topology + 房间层输出 |

房间层确定 → 机电层才能定位；机电层反馈约束时（如管件超出边界）→ 由 AG2-1 重新计算房间层。
两层之间通过 `geo`（坐标原点、SCALE_MAIN、hdr_y 等）传递。

## 4. 竖向剖面图规则 (Section View)
展示设备沿 Z 轴的竖向空间关系：
1. **基准划分**：绘制一条横向粗线代表 **`Z-top`** 楼板。
2. **地下区域 (Z < Z-top)**：绘制集水池与集水坑轮廓。水泵放置于坑底。根据输入标高，用蓝色水平虚线与半透明填充绘制各级水位（停泵、启泵等）。
3. **地上区域 (Z > Z-top)**：绘制维护间空间（高度为 `h_room`）。压水管从水泵垂直向上，穿过 `Z-top` 楼板后，在指定出水标高转为水平，绘制管道和阀门侧视或截面。
4. **代表支路**：剖面图只绘制一条代表泵支路，默认取拓扑排序后的第一台泵；全部泵并联数量和支路关系由平面梳齿图表达。
5. **拓扑一致性**：代表支路上的阀门/仪表来自 `buildLayoutMap.representativeBranch.items`，剖面图不得使用固定漂浮阀门替代拓扑设备链。
6. **集水坑高度**：集水坑竖向深度按 `H_sump = Z_bottom - Z_sump` 表达；当前 AG1-1 中 `Z_stop = Z_bottom`，因此渲染时使用 `Z_stop - Z_sump`。

## 5. 自动化管件与拦截器 (If-Else)
遍历拓扑连线时，利用几何特征自动插入过渡件占据物理空间：
1. **大小头 (Reducer)**：`IF (设备 DN != 管道 DN)`。在两者间强制渲染梯形变径符号，并累加物理长度，推移后续相连设备。
2. **弯头 (Elbow)**：`IF (管道方向空间转折，如剖面图穿过 Z-top 后转水平)`。插入弯头，其半径占用空间。
3. **三通 (Tee)**：支管汇入梳脊总管的交叉点，强制绘制三通符号，禁止线条十字穿透。

## 6. 视觉规范与自动标注
* **色彩体系**：水泵 `#2471a3`，管道 `#2980b9` (支管细,主管粗)，阀门 `#c0392b`，集水池底色 `#f8fbff`。
* **自动尺寸线**：
    * 平面图外围：标注机房总长 `L`、总宽 `W`、泵间净距 `d`。
    * 剖面图旁侧：使用三角符号标注 `Z-top` 及各水位绝对标高（如 `▽ 4.500`）。
