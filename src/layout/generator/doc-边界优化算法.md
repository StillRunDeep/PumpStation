# AG4 建筑平面布局 2：阶段三 - 边界优化与空隙填充算法

## 1. 阶段定位与核心目标

在“约束生长算法”（见 `AG4建筑平面布局1_约束生长算法.md`）完成了**阶段 1（全局矩形扩展）**和**阶段 2（L/U形填补扩展）**之后，建筑轮廓内通常仍会遗留部分未被占据的零散空隙（Gaps）。同时，部分房间在阶段 2 中为了达到面积目标，可能形成了利用率较低（如包围盒极大但实际面积较小）的复杂多边形边界。

因此，**阶段 3（边界优化与空隙填充）**作为布局生成的最后收尾环节，承担着两个核心目标：
1. **空隙完全分配**：将建筑轮廓内阶段 2 遗留的所有空隙（Empty Cells）完全分配给相邻房间，确保建筑内部无死角。
2. **边界重新调整（优化利用率）**：借此机会重新评估并平滑相邻房间之间的交界线。通过消除凹角、拉直边界、甚至在必要时微调已分配的网格归属，使得房间的轮廓更加方正，显著提升**房间利用率（实际占据面积 / 包围盒面积）**并降低多边形顶点数。

## 2. 核心方案：线段优先填充与边界平滑 (Boundary-First Filling & Smoothing)

该算法以**连续的线段（Segment）**而非单个格子作为操作基元，通过启发式评分来做出最有利于全局边界简化的决策。

### 2.1. 算法流程

在每一次迭代中，算法执行以下步骤，直至所有目标达成：

1. **识别待处理区域**：
   - 找出当前所有最长的、连续的水平和垂直**空白线段**。
   - 识别相邻房间之间错综复杂、导致双方利用率低下的**锯齿状交界线**（用于边界微调）。
2. **评估与评分（两阶段混合评分法）**：对每个候选线段或交界线应用评分规则，评估将其分配给哪个房间能带来最大的形状收益。
3. **选择最优并执行**：选择全局得分最高的修改方案（填充空隙或交换边界网格），执行状态更新。
4. **重复**：回到步骤 1，直至没有空白格子，且边界利用率达到局部最优。

### 2.2. 两阶段混合评分法

#### 阶段 A：优先处理“凹角”与狭长缝隙 (消除明显缺陷)

在进行常规评分前，算法会首先扫描并处理掉所有**最高优先级**的明显形状缺陷。
- **识别模式**：寻找被**同一个房间**从三个方向（或两个长边和一个短边）深度包围的空白线段，或者极其狭长的夹缝。
- **操作**：立即将这些线段分配给包围它的房间。
- **目的**：这相当于在“填补房间的凹角”，是确定性最强、收益最高的简化操作。能快速让房间形状变得“饱满”，迅速提升该房间的**利用率指标**。

#### 阶段 B：以“提高利用率”与“减少顶点”为目标的量化评分

对于剩余的空隙和可调整的边界，算法进行量化综合评分：
- **核心指标**：直接量化一次操作对“房间利用率”和“边界复杂度（顶点数）”的改善程度。
- **分数计算**：
  1. 假设将候选线段分配（或重新划分）给邻居 `room_A`。
  2. **顶点收益**：计算此操作**消除**的旧顶点数减去**创造**的新顶点数（`ΔVertices`）。
  3. **利用率收益**：计算此操作后，`room_A` 的（实际面积 / 包围盒面积）的提升比率（`ΔUtilization`）。
  4. `Score = (α × 顶点收益) + (β × 利用率收益)`。（注：α/β 系数**当前实现先忽略**，待后续在 `scorer-params.js` 中统一管理）
- **决策**：
  - 对所有可能的组合计算分数，选择能提供**最高正分**的组合执行。
  - **决胜局 (Tie-Breaker)**：分数相同时，优先选择更长的线段，或优先补偿当前利用率最低的房间。

## 3. 边界重新调整（二次优化）机制

除了单纯的“填空”，阶段 3 的高级特性在于对既有边界的**二次调整**。为了确保优化过程不会牺牲功能性，需引入以下约束：

- **锯齿消除**：当两个房间的交界线呈现阶梯状（多个内角和外角交替）时，算法可识别这些导致双方包围盒剧增的“锯齿”网格，将其统一划拨给其中一方，从而拉直交界线。
- **面积与形状置换（受控调整）**：允许房间 A 退让部分突出的网格给房间 B。这种微调能够让双方的形状都更加接近矩形，双双提高利用率得分。
- **小房间保护机制（Small Room Guard）与弹性置换**：
    - **面积波动容忍度**：在边界调整中，允许房间面积在目标值的 **[95%, 110%]** 范围内动态波动。大房间由于基数大，承担更多的填充义务；小房间则在保证功能面积的前提下，参与边界置换。
    - **弹性边界置换（Give-and-Take）**：边界优化不是单向的侵蚀，而是双向的置换。算法支持“滑动边界”操作——例如小房间 A 在西侧退让网格给房间 B 以拉直大房间边界，作为交换，房间 B 在北侧补偿同等面积给 A。只要这种置换能显著提升两个房间的**综合利用率（Util_A + Util_B）**，即视为高质量优化。
    - **形状优先原则**：置换的触发条件应以“形状去畸”为核心。如果一个小小的“此消彼长”能让两个相邻房间同时从 L 形回归为矩形，算法将赋予该操作极高的评分权重。
    - **拓扑连通性底线**：在动态置换过程中，严禁出现将任何房间”切断”或导致其不闭合的操作。
- **走廊最小宽度守护**：`corridor_l1` 在任意横截面（行或列）上的宽度不得低于 **3 格（1500mm）**。在空隙填充阶段，若 `corridor_l1` 当前最小宽度不足 3 格，则相邻空白格子优先保留给走廊，不得被其他房间占用。

## 4. 方案优势

- **完美收尾**：彻底解决约束生长法中因早期随机性或碰撞导致的微小空隙问题。
- **目标驱动（利用率导向）**：不仅填满空间，更主动追求更高的“房间利用率”和更低的“形状复杂度”，直接与后端的方案评分（Scorer）标准对齐。
- **宏观视角**：以“线段”作为操作单元，从宏观上保证了边界的连续性与平直性。
- **视觉与实用性双优**：产生的平面图更接近人类设计师的直觉，墙体平直，大幅减少了施工复杂度和死角浪费。
