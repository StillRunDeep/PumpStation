# AG4 建筑平面布局阶段三 - 边界优化与空隙填充算法

## 1. 阶段定位与核心目标

在"约束生长算法"（见 `doc-约束生长算法.md`）完成了**阶段 1（全局矩形扩展）**和**阶段 2（L/U形填补扩展）**之后，建筑轮廓内通常仍会遗留部分未被占据的零散空隙（Gaps）。同时，部分房间在阶段 2 中为了达到面积目标，可能形成了利用率较低的复杂多边形边界。

**阶段 3（边界优化与空隙填充）**承担两个核心目标：
1. **空隙完全分配**：将建筑轮廓内阶段 2 遗留的所有空隙完全分配给相邻房间。
2. **拐角消减（Phase 3c）**：通过凹角驱动的边界交换，最小化各房间的多边形顶点数。

---

## 2. Phase 3c 拐角消减交换（`runAreaSwap`）——修订设计

### 2.1 设计背景：原方案的根本缺陷

原方案要求每次操作**面积严格中性**（填入 k 格 = 回退 k 格），实测发现三类系统性失效：

| 症状 | 根因 |
|------|------|
| 净顶点反而增加 | 回退格从 A 的平整边界取出，产生新凹角，抵消甚至超过填入的顶点收益 |
| 回退格为零 | A 的所有邻居都是填入来源，无第三方接收回退格 |
| 来源房间断裂 | 段过长，来源房间失去连通或面积偏差过大 |

根本矛盾：**形状优化与面积中性是两个相互干扰的目标**，强行在单步内同时满足，导致两个目标都无法达成。

### 2.2 新设计：形状优化与面积修正解耦

将拐角消减拆为两类独立操作，交替执行：

```
每轮迭代：
  1. 优先执行 ShapeFill（形状填入）：有效时执行，面积可临时偏大
  2. 无 ShapeFill 可用时，执行 AreaCorrect（面积归位）：无效时收敛退出
```

**防振荡机制（方案 C）**：ShapeFill 要求 A 自身顶点**严格减少**（≥ 2）才触发。由于每次 ShapeFill 都减少 A 的顶点，而顶点数有下界（矩形=4），ShapeFill 天然收敛，不会振荡。

---

### 2.3 操作一：ShapeOp（形状操作——填入或裁剪）

**目标**：在每个凹角处，比较 ShapeFill（A 吸收邻居格）与 ShapeTrim（A 让出自身格），选取 A 顶点减少更多且满足约束的方案，消除 A 的凹角。

**触发条件**：操作后 A 自身顶点严格减少 ≥ 2。

**流程**：

**① 生成候选操作（`findShapeOperations`）**

1D 段（单行/列）不足以消除凹角，L 形房间的凹陷是 **2D 矩形区域**。

对每个 count=3 凹角的缺失格 `(mx,my)`，以 A 的 bbox 为界枚举 **4 个象限矩形**，每个象限同时生成两类候选：

| 操作类型 | 操作区域 | 含义 |
|----------|----------|------|
| **ShapeFill** | 象限矩形内的非 A 格 | A 吸收邻居格，填平凹陷；A 面积增大 |
| **ShapeTrim** | 互补矩形内的 A 格（bbox 同列/行另一侧） | A 让出自身格给邻居，裁掉凸出臂；A 面积减小 |

示例（右下象限 ）：
- Fill 区： 内非 A 格（A 吸收）
- Trim 区： 内 A 格（A 让出，为同列对侧）

Trim 格的接收方：对每个 trim 格，从其四邻中选一个 非 A 的邻居房间接收（优先面积最不足的邻居）。

**② 评估并选最优**

对每个候选操作，在 `tempGrid` 上模拟，检查：

| 检查项 | ShapeFill | ShapeTrim |
|--------|-----------|-----------|
| A 自身顶点减少 | ≥ 2 | ≥ 2 |
| 来源/接收房间连通 | BFS 验证 | BFS 验证 |
| 来源房间面积偏差（Fill）| ≤ 30% | — |
| 接收房间面积偏差（Trim）| — | ≤ 30% |
| A 连通 | 验证 | 验证 |

选全局 **A 顶点减少最多** 的候选执行（Fill 与 Trim 公平比较）。

**③ 执行操作**

- ShapeFill：将候选格从各来源房间转入 A，A 面积增加。
- ShapeTrim：将候选格从 A 转出给各接收邻居，A 面积减少。

**约束汇总**：

| 约束 | 说明 |
|------|------|
| A 顶点减少 ≥ 2 | 防振荡核心；保证每次操作单调改善 A 的形状 |
| 来源/接收房间连通 | BFS 验证 |
| 来源/接收房间面积偏差 ≤ 30% | 保护邻居房间不被过度抽空或撑大 |
| A 面积软约束 | Fill 后可临时超标由 AreaCorrect 修正；Trim 后面积减小 |

---

### 2.4 操作二：AreaCorrect（面积归位）

**目标**：将面积超标的房间 A 的多余格子让给邻居，同时不产生新凹角。

**触发条件**：任意房间 A 的实际格数超过 `targetGridCount × (1 + PHASE3_AREA_DEVIATION)`（即超标 15%）。

**流程**：

**① 识别超标房间**

扫描所有房间，找出面积超标房间按超标量从大到小排序。

**② 寻找可让出的格子**

从超标房间 A 的边界格中，逐格（或逐批）验证：在 `tempGrid` 上将该格让给邻居 B 后，**所有受影响房间（A + B）的总顶点不增加**。

需要同时满足：
- A 移出该格后顶点不增加（该格是 A 的"凸出格"）
- B 接收该格后顶点不增加（该格恰好填补 B 的凹角，或落在 B 的平整边界上）

若 B 接收后顶点增加，则换其他邻居或跳过该格。

**③ 择优让出**

在所有通过验证的（格子, 邻居）组合中，选全局总顶点减少最多（或至少不增加）的方案执行，优先让给面积不足的房间。

**约束汇总**：

| 约束 | 说明 |
|------|------|
| 全局顶点不增加 | `Σ countRoomVertices(所有受影响房间) 后 ≤ 前` |
| A 连通 | BFS 验证 |
| 接收方 B 连通 | BFS 验证 |
| 面积单调收敛 | A 每次 AreaCorrect 后面积减少，保证收敛 |

---

### 2.5 迭代控制与收敛

```
runAreaSwap(grid, rooms):
  MAX_ITER = 50
  for i in 0..MAX_ITER:
    best = findBestShapeFill(grid, rooms)   // A 顶点减少 ≥ 2
    if best:
      apply(best)
      continue
    corrected = runAreaCorrect(grid, rooms) // 面积超标归位
    if corrected:
      continue
    break  // 两者均无可用操作，收敛
```

**收敛保证**：
- ShapeFill 每次减少某个房间的顶点 ≥ 2，顶点总数有下界（所有房间矩形时为 4N），故 ShapeFill 最多执行有限次
- AreaCorrect 要求全局顶点不增加 + A 面积减少，面积单调收敛到目标范围，顶点不反弹
- 两者均不产生振荡：ShapeFill 不减少 A 自身顶点则不执行；AreaCorrect 全局顶点增加或面积不减少则不执行

**来源房间亏空的修复路径**：ShapeFill 执行后来源房间 B 面积亏空，通过两条路径自然修复：
1. A 执行 AreaCorrect 时优先将多余格让给面积不足的 B（若满足全局顶点不增加约束）
2. 若 A→B 不满足约束，B 在后续迭代中触发自身的 ShapeFill，从其他相邻房间补充面积

**面积超标的软约束**：若 AreaCorrect 找不到满足全局顶点不增加的有效移动，A 的超标面积保留，算法静默收敛退出。面积偏差作为软约束接受，不强制归零。

---

### 2.6 前置概念

**count=3 凹角格点**：以 2×2 方格邻域扫描，若某格点 P 的四个相邻格子中恰好有 3 格属于房间 A，1 格（缺失格 `(mx,my)`）属于其他房间，则 P 是 A 的一个内凹角。

```
A A
A .  → P 在 A、A、A、. 的交叉点，count=3
```

**`countRoomVertices(grid, roomId)`**：用 2×2 格点扫描统计房间的多边形顶点数。矩形=4，L形=6，U形=8。

### 2.7 关键辅助函数

**`findConcaveCorners(grid, roomId)`**：扫描房间 bbox，返回所有 count=3 凹角格点及其缺失格坐标和来源房间 id。

**`findConcaveEdgeSegments(grid, roomId)`**：对每个凹角的缺失格，在所在行/列向两端延伸，返回所有连续非 A 格段（去重）。

**`applyShapeFill(tempGrid, roomId, fillCells)`**：将 fillCells 从各自 ownerId 移入 roomId，不执行回退。

**`findAreaCorrectMoves(grid, roomId, roomMap)`**：枚举 A 边界格让给各邻居的所有组合，在 tempGrid 上验证全局顶点（A + 邻居）不增加，返回按全局顶点减少量排序的候选列表，供 AreaCorrect 使用。

## 4. 原始设计方案（参考）

### 4.1 线段优先填充与边界平滑（Boundary-First Filling & Smoothing）

以**连续的线段（Segment）**而非单个格子作为操作基元，通过启发式评分决策。

**算法流程（每次迭代）：**
1. 找出最长的连续水平/垂直空白线段，以及导致双方利用率低下的锯齿状交界线。
2. 对每个候选线段应用评分规则：`Score = (α × 顶点收益) + (β × 利用率收益)`。
3. 选全局最高正分方案执行。

### 4.2 二次边界调整约束

- **锯齿消除**：将阶梯状交界线统一划拨给一方，拉直边界。
- **弹性置换（Give-and-Take）**：A 在西侧退让给 B，B 在北侧补偿 A，双向置换显著提升综合利用率时触发。
- **面积波动容忍度**：`[95%, 110%]` 目标面积范围内允许动态波动。
- **拓扑连通性底线**：任何操作严禁切断房间。
- **走廊最小宽度守护**：`corridor_l1` 任意截面宽度 ≥ 3 格（1500mm）。
