# 泵站业务

> 模块版本：v2.0（含空间临近关系与交通组织）

---

## 一、模块目标

自动生成多种完整建筑平面布局方案，供用户选择。每种方案须满足：

- 所有功能空间正确放置在对应楼层
- 有外部直接进入需求的房间必须临近外墙
- 起重机覆盖范围约束满足
- **功能关联房间通过走廊或大厅相互连通**

---

├─────────────────────────────────────────────────────────────────┤

│  变压器房间  停车间  水表间  消防间  泵房维护间  钢梯            │
│                      吊装孔位            进风井  排风井(东侧)   │

### 2.2 关键竖向对位关系

> 注：`dock1` 与 `dock2` 在平面上**无需**相互对齐；低压配电室与屋面设备间**无需**竖向对齐。以下对位关系为设备/管线连接需求，不作为平面布局位置约束。

| 对位组       | 涉及楼层                           | 说明                                                        |
| --------- | ------------------------------ | --------------------------------------------------------- |
| **吊装竖井**  | 一层 ↔ 地面层                       | `dock1`（地面层）和 `dock2`（一层）分别在各楼层提供吊装孔洞，两者平面位置**无需**对齐；设备经各楼层独立的吊装口进出 |
| **楼梯/钢梯** | 屋面 → 一层 → 地面层 → 埋地层            | 楼梯贯穿一层至屋面；钢梯自地面层向下至埋地层（可不连通为一条竖向通道，出于消防安全）                |
| **进风系统**  | 地面层进风井 ↔ 一层进风窗                 | 进风井在地面层东侧外墙内，向上连通一层进风窗；新风由此进入风机房                          |
| **排风系统**  | 一层风机房 → 屋面排风区 ↔ 地面层排风井 → 底层排风窗 | 风机房向上排风至屋面；同时向下通过排风井至湿井排风窗，用于水池通风                         |
| **LV配电室** | 一层配电间 ↔ 屋面配电间/空调区              | 低压配电室设备延伸至屋面（管线连接），屋面设空调机组及进排风设备；两者平面位置**无需**竖向对齐          |

### 2.3 本模块（AG4-2）覆盖范围

AG4-2 当前仅生成**地面层**和**一层**的平面布局，其余层次说明如下：

| 楼层    | AG4-2 是否处理 | 说明                          |
| ----- | ---------- | --------------------------- |
| 屋面    | ✗          | 优先级较低，设备布置待后续               |
| 一层    | ✓          | 完整生成平面                      |
| 地面层   | ✓          | 完整生成平面                      |
| 埋地层   | ✗          | 由 AG1-1/AG3-1 处理（水池深度/泵室剖面） |
| 底层/湿井 | ✗          | 由 AG1-1 处理                  |

### 3.1 必须临近（MUST ADJACENT）

| 编号   | 房间 A              | 房间 B            | 原因                                |
| ---- | ----------------- | --------------- | --------------------------------- |
| M-01 | 总水表房 `meter_main` | 水表房 `meter_sub` | 两间水表房并排，共用同一外墙区段，便于计量管理           |
| M-02 | 变压器房1 `trafo1`   | 变压器房2 `trafo2`  | 两间变压器房相邻布置，方便母线桥架连接；**两房间共享侧墙但不开内门** |



### 3.2 建议临近（SHOULD ADJACENT）

满足则加分，不满足不影响方案可行性。

| 编号   | 房间 A               | 房间 B               | 原因                           |
| ---- | ------------------ | ------------------ | ---------------------------- |
| S-01 | 低压配电室 `lv_control` | 一层走廊 `corridor_l1` | 控制室须通过内走廊进入，避免直接对外开门         |
| S-02 | 清洁泵房 `clean_pump`  | 一层走廊 `corridor_l1` | 通过走廊进出，兼做管道检修通道              |
| S-03 | 雨水回收房 `rainwater`  | 一层走廊 `corridor_l1` | 同上                           |
| S-04 | 风机房 `fan_room`     | 一层走廊 `corridor_l1` | 风机房应通过走廊连通，避免孤立              |
| S-05 | 风机房 `fan_room`     | 送货口2 `dock2`       | 设备经吊装口就近进出风机房，距离越近越便利        |
| S-06 | 消防设备房 `fire_equip` | 水表房 `meter_sub`    | 小型服务用房宜集中布置，减少外墙开门分散         |
> **小型服务用房集中原则**：`meter_main`、`meter_sub`、`fire_equip` 的外门位置应尽量集中在同一外墙段，而非分散布置于建筑各侧。

### 3.3 Dock 2 位置说明

`dock2`（一层吊装孔位）不要求设置于风机房内部，改为：

- 可位于**一层走廊 `corridor_l1`** 区域内，或走廊与风机房相邻的区域；
- 须处于 **5t 单轨吊覆盖范围（`crane5`）** 内；
- 风机房 `fan_room` 宜靠近 `dock2`（见 S-05）。

## 四、交通组织（走廊/大厅）

### 4.1 地面层

| 空间           | 说明                                         |
| ------------ | ------------------------------------------ |
| 停车间+ 泵房维护间   | 核心交通空间，15t 桥式起重机覆盖全区。设备经由卡车停泊→送货口1→泵室竖向进出。 |
| 小型服务房（水表、消防） | 沿外墙排列，各自独立对外门，无需内廊。                        |
| 变压器房         | 直接对外设检修门，无内部流通需求。                          |

> 地面层各房间均有外门，**不需要专用走廊**。维修大厅本身即为主要交通空间。

### 4.2 一层

一层设 **主走廊 `corridor_l1`**（最小净宽 1500 mm），连通以下房间：

```
[低压配电室] ←→ [走廊] ←→ [清洁泵房]
                  ↕             
             [雨水回收房]   [风机房] ←→ [dock2（走廊内/走廊侧）]
```

走廊位置：位于低压配电室与清洁泵/雨水回收房之间；风机房亦应与走廊连通（见 S-04），`dock2` 可布置于走廊区域内（见 3.3）。

| 参数   | 数值                  |
| ---- | ------------------- |
| 最小净宽 | 1500 mm             |
| 典型净高 | ≥ 2400 mm           |
| 典型位置 | 低压配电室西/东侧，纵向贯通一层南半部 |

## 五、门的放置规则

### 5.1 连接原则

房间之间能否设门，取决于是否存在连接需求：

| 情形                               | 是否可开门  |
| -------------------------------- | ------ |
| 两房间存在 MUST 或 SHOULD 临近关系         | ✓ 可开内门 |
| 房间与交通空间（`corridor_l1` 或维修大厅）相邻   | ✓ 可开内门 |
| 地面层房间有 `ext_access` 约束，接触外墙（非南墙） | ✓ 可开外门 |
| 两房间无任何连接规则关系                     | ✗ 不得开门 |

**门数量约束**：每个房间**仅设一扇门**（内门或外门，二选一），不得同时设内门和外门。

**特殊：`trafo1` 与 `trafo2`** 虽为 MUST 邻接，两房间共享侧墙，但**不开内门**（有共享墙，无门洞）。外部检修通过各自外墙门进入。

### 5.2 门的位置

- **内门**：在两房间共用边线（shared wall segment）的重合段内，将门放置在靠近端部的位置（取重合段长度的 1/4 处），以**远离**两房间的各自中心点。
- **外门**：在房间接触外墙的墙段内，同样靠近墙段端部放置。
- 标准门宽：900 mm（单扇）。

### 5.3 特殊情况

- **一层**：所有房间均**不直接对外墙开门**，须经走廊 `corridor_l1` 进出（包含风机房）。
- **风机房** `fan_room`：体量大，可在**走廊侧**设检修门；虽可接触北墙/西墙，但**一层不应设置外门**。
- **吊装口** `dock1` / `dock2`：为楼板开口，不设门。

## 六、平面布局方案逻辑描述

### 基于**约束生长法**（见 `AG4建筑平面布局1_约束生长算法.md`）对各功能房间进行随机初始位置生长，最终输出多个方案。

#### 6.0 用户交互参数

在运行前向用户展示以下可调参数，用户确认或修改后方可进入生长算法。

##### 6.1 建筑整体尺寸

| 参数            | 初始值   | 说明                              |
| ------------- | ----- | ------------------------------- |
| 建筑宽度BW-东西向距离 L（mm） | 43850 | **最大值约束**：实际布局可小于此值；用户可修改；修改后缓存 |
| 建筑进深BD-南北向距离 W（mm） | 18600 | **最大值约束**：实际布局可小于此值；用户可修改；修改后缓存 |

> - L、W 为建筑外包尺寸的上限，实际生成方案中各功能空间总和决定占地面积，可小于 L × W。  
> - 建筑形状可为**方形或 L 形**，不限于矩形轮廓。  
> - 系统同时以相同基底面积（`S = L × W`）自动推算多种长宽比变体（见 A.6）；各功能房间的长宽比在生长过程中允许随机变化，不固定比例。  
> - 图纸按照上北下南进行方位对应。

##### 6.2 功能空间目标面积

以下为各功能空间目标面积的初始值，用户可逐项修改。标注 `[继承]` 的参数值来自前序模块计算结果，修改前系统给出提示。

**地面层（Ground Floor）：**

| 空间      | 标识            | 目标面积初始值（m²） | 最小面积（m²）  | 来源说明                       |
| ------- | ------------- | ----------- | --------- | -------------------------- |
| 变压器房 1  | `trafo1`      | 60          | 42        | 固定经验值；可布置于东/西/北侧            |
| 变压器房 2  | `trafo2`      | 60          | 42        | 固定经验值；可布置于东/西/北侧            |
| **停车位** | `parking`     | 150         | 150       | **最小 150 m²**，满足 5.5t 货车停放需求 |
| 泵房维护间   | `repair_zone` | 120         | 120       | **最小 120 m²**，满足三条 DN1000 主管布置 |
| 总水表房    | `meter_main`  | 12          | 8         | 固定经验值                      |
| 水表房     | `meter_sub`   | 8           | 6         | 固定经验值                      |
| 消防设备房   | `fire_equip`  | 15          | 10        | 固定经验值                      |

**一层（First Floor）：**

| 空间            | 标识            | 目标面积初始值（m²） | 最小面积（m²）  | 来源说明                |
| ------------- | ------------- | ----------- | --------- | ------------------- |
| 低压配电及 PLC 控制室 | `lv_control`  | 65          | 45        | 固定经验值               |
| 风机房           | `fan_room`    | 55          | 38        | 固定经验值               |
| 清洁泵房          | `clean_pump`  | 25          | 17        | 固定经验值               |
| 雨水回用泵房        | `rainwater`   | 25          | 17        | 固定经验值               |

> - 各房间「最小面积」= 目标面积初始值 × 70%（走廊及有硬性下限的房间除外）。  
> - `parking` 和 `repair_zone` 的最小面积为硬性约束，不适用 70% 规则。

> - **提示规则**：若某面积值来自上游模块（标注 `[继承]`），系统提示"此值由前序模块计算得出，修改后不再与上游同步，确认修改？"用户确认后可修改，不回写上游。

#### 6.4 算法流程（每次随机初始化）

```
输入：用户确认的建筑尺寸 L×W、各功能空间目标面积、约束声明
输出：一个完整的地面层 + 一层平面布局

Step 0  用户交互确认（仅首次运行时执行，后续重复生长复用参数）
        展示 A.0 中的所有可调参数及其初始值；
        对继承自上游模块的参数给出提示；
        等待用户确认或修改，锁定 L、W 及各房间目标面积。

Step 1  栅格化
        将 L×W 建筑轮廓映射为 (L/100)×(W/100) 网格；
        按比例标记 crane15、crane5 区域边界。

Step 2  随机化决策
        ① 随机选择各个功能房间的种子位置

Step 3  权重网格生成
        - 有外墙约束的房间：外墙 100mm 内权重 = 10，其余权重 = 1
        - 有邻接约束的房间：已占用邻居周围 200mm 内权重提升 × 3
        - 起重机覆盖区域以外的格子：crane 约束房间权重置 0

Step 4  房间放置（Room Placement）
        按面积比例（大房间优先）逐一选取加权随机初始位置；
        parking 与其他功能空间同等参与竞争；
        若邻接约束未被满足，则重置重试（最多 20 次）。

Step 5  阶段 1 — 全局矩形扩展
        所有房间优先且仅允许进行矩形扩展，直到达到极限。

        ▼ 检查点 A（硬性功能红线过滤门）
Step 5A 矩形扩展结束后，对当前网格快照调用评价.md **第一梯队**评价函数，逐方案评价：
        • computeMissingRoomsPenalty()   —— 缺失房间 (−500/间)
        • computeDoorAccessPenalty()     —— 可达性违规 (−300/间)
        • violations 计算                —— MUST 约束违反 (−200/项)
        通过条件：三项惩罚之和 = 0（即硬性红线全部满足，无任何扣分）。
        - 通过 → 方案进入 Step 6（L/U 形扩展）
        - 未通过 → 丢弃该方案，重新执行 Step 2–5 生成替代方案
        - 持续循环，直至凑齐 9 个通过方案，再整体进入 Step 6

Step 6  阶段 2 — L/U 形填补扩展
        9 个通过检查点 A 的方案，继续进行非矩形扩展，填补剩余空间（受 8 顶点形态约束）。
        【输出要求】本步骤执行结果（各房间最终坐标与尺寸）须完整体现在
        LayoutVariant.placements 及 roomDimensions 字段中。

        ▼ 检查点 B（硬性红线 + 空间质量排序）
Step 6A L/U 形扩展结束后，对当前网格快照调用评价.md **第一梯队 + 第二梯队**评价函数：
        第一梯队（同检查点 A）：missingRooms、doorAccess、violations
        第二梯队：
        • computeAspectRatioPenalty()    —— 形状与比例 (−100/单位违规)
        • computeSpaceEfficiency()       —— 空间有效率 (+150 pt 上限)
        • corridor 完整性                —— 走廊连通房间数 (+80 pt 上限)
        对 9 个方案按上述指标的部分合计得分（`_checkpointBScore`）降序排列，供后续处理参考。
        （无淘汰逻辑，仅排序）

Step 7  阶段 3 — 边界优化与空隙填充（暂跳过，后续补充）
        消除锯齿，平滑相邻房间交界线，优化房间综合利用率。

Step 8  连通性处理（Room Connectivity）
        ① 显式连通约束（MUST 邻接对）优先放门；
        ② 走廊 corridor_l1 连接所有相邻房间；
        ③ 可达性检验：若有房间孤立，沿邻接路径补门。
        门位置：共用墙段的 1/4 处（远离房间中心，见 Section 五）。

Step 9  转化为正式视图 → 全量评价
        正式视图生成后，调用 scoreLayout()（三梯队全量评价函数）：
        第一梯队：missingRooms、doorAccess、violations
        第二梯队：aspectRatio、efficiency（空间有效率）、corridor（走廊完整性）
        第三梯队：adjacency、accessibility、trafo、fanRoom、growthSuccess
        输出完整 score / spaceEfficiency / breakdown，供 UI 展示排名与得分明细。
```

#### 6.5 多方案生成与进化筛选

- **持续生成逻辑**：系统提供”生成更多方案”功能，通过后台异步循环不断探索更优布局。
- **进化策略**：为了防止陷入局部最优并保留优秀基因，每轮生成采用 **”2 个交叉进化 + 7 个随机探索”** 的混合配比。
- **亲本选择规则**：
  - 第一组：**第 1 名 ↔ 第 5 名** 交叉，利用顶尖方案与中坚方案结合。
  - 第二组：**第 5 名 ↔ 第 9 名** 交叉，利用中后部方案增加多样性。
  - 这种跨排名的配对方式能有效降低第一名方案的”统治力”，防止种群过早收敛（基因单质化）。
- **分阶段过滤**：每轮生成在矩形扩展（阶段 1）结束后即执行**检查点 A**（硬性功能红线评价）。
  - 未通过的方案立即丢弃，触发重生成逻辑（随机或交叉遗传），最多尝试 50 次。
  - 仅当凑齐 9 个通过检查点 A 的方案后，才整体进入 L/U 形扩展（阶段 2）。
  - 阶段 2 结束后执行**检查点 B**（第一 + 第二梯队评价），对 9 个方案排序，为后续处理提供优先级参考。
- **实时筛选与排序**：
  - 合并新旧方案，优先保留 `violations = 0` 的可行方案。
  - 始终保留得分最高的前 **9 个** 候选方案。
  - 网页界面实时更新排序，发现更优方案时给出动态提示。

## 九、输入输出定义

**输入**：

```ts
interface UserParams {
  buildingW: number        // 东西向宽度 BW（mm），最大值约束，默认 43850
  buildingD: number        // 南北向进深 BD（mm），最大值约束，默认 18600
  roomTargetAreas: {       // 各功能空间目标面积（m²），用户可修改
    trafo1: number         // 默认 60
    trafo2: number         // 默认 60
    parking: number        // 默认 150（最小值，满足 5.5t 货车停放需求）
    repair_zone: number    // 默认 120（最小值，满足三条 DN1000 主管布置）
    meter_main: number     // 默认 12
    meter_sub: number      // 默认 8
    fire_equip: number     // 默认 15
    lv_control: number     // 默认 65
    fan_room: number       // 默认 55
    clean_pump: number     // 默认 25
    rainwater: number      // 默认 25
    corridor_l1: number    // 0（走廊面积不作要求，宽度须 ≥ 1500 mm）
  }
}
```

**输出**：

```ts
interface LayoutVariant {
  id: string               // 方案编号（S-1...）
  buildingSize: { L: number; W: number }  // 实际使用的建筑尺寸 (mm)
  groundPlacements: Placements // 各房间坐标、尺寸及占据的网格数据
  level1Placements: Placements
  doors: DoorPlacement[]   // 门的位置与类型
  violations: Violation[]  // 生成阶段未满足的 MUST 约束列表
  feasible: boolean        // 是否满足所有硬性约束
  // 注：评分明细 (breakdown) 与综合得分 (score) 由 AG4-3 评价模块补充，见相应文档。
}
```
