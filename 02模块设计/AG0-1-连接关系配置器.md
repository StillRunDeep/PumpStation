# AG0-1 连接关系配置器

> 功能分类：**用户交互（输入）+ 解析代理（输出）**
> 管道对象命名与阀件定义见 [00-计算定义总纲.md](./00-计算定义总纲.md)

---

## 一、职责概述

AG0-1 分两部分：

| 部分 | 位置 | 说明 |
|------|------|------|
| **拓扑编辑器** | 用户输入面板（"设备拓扑配置"折叠块） | 交互控件，用户定义设备连接关系和房间归属 |
| **拓扑解析代理** | 结果面板 AG0-1 卡片 | 点击"开始计算"后运行，对拓扑做连通性检查和统计 |

系统根据 N 和 N_spare 自动生成默认拓扑，用户可编辑后点击"开始计算"，AG0-1 解析拓扑并输出结构化结果供 AG3-1 使用。

---

## 二、在系统中的位置

```
用户输入面板
  ├── Q、N、S 表单
  ├── 高级参数（折叠）← N_spare 备用泵台数（默认 0）
  └── 设备拓扑配置（折叠，默认展开）← AG0-1 编辑器
          ↓ 开始计算
AG0-0：参数验证（Q/N/S）
AG0-1：拓扑解析  → 输出 Topology
  │
  ▼
AG3-1：按 roomId 渲染设备位置
```

## 三、默认拓扑结构

以 N=2、N_spare=0 为例（默认无备用泵）：

```
进水（节点）
  ├──► P1（泵）──► 止1 ──► 闸1 ──┐
  ├──► P2（泵）──► 止2 ──► 闸2 ──┤──► 汇（节点）──► 总闸 ──► 流量计 ──► 出水（节点）
  │                                    └──► 旁闸 ──► 旁止 ──► 进水（旁通回水）
  └── N / N_spare 实时响应输入框变化，拓扑自动重置
```

若 N_spare=1，则额外生成一行备用泵支路（标签"备"），接入同一汇流节点；
N_spare=2 则生成"备1"/"备2"，以此类推。AG12 泵房尺寸按实际 N_total=N+N_spare 计算。

- 水泵默认放在**集水坑（wet_well）**，止回阀/闸阀在**泵房维护间**
- 各支路汇聚到汇流节点，再经总闸、电磁流量计到出水口
- 旁通管从汇流节点分出，经旁闸、旁止回水到集水坑

## 四、房间定义

| 房间 ID | 名称 | 画布位置 |
|---------|------|----------|
| `wet_well` | 集水坑 | 左侧，蓝色虚线框 |
| `pump_room` | 泵房维护间 | 右侧，浅蓝实线框 |

设备拖入不同房间后自动更新 `roomId`，AG3-1 据此决定渲染位置。

## 五、数据模型

```ts
interface Topology {
  rooms:   Room[]
  nodes:   Node[]     // 连接点（设备接口 or 独立节点）
  devices: Device[]   // 设备（通过 nodeIds 挂载到节点）
  pipes:   Pipe[]     // 线（连接 nodes，无向）
}

// 连接点（节点）
interface Node {
  id:       string   // 唯一 ID，如 "node_1"
  label:    string   // 如 "进水"、"出水"、"汇"
  canvasX:  number   // SVG 画布坐标
  canvasY:  number
}

// 设备
interface Device {
  id:       string
  type:     'pump' | 'check_valve' | 'gate_valve' | 'flowmeter'
  label:    string
  nodeIds:  string[]        // 该设备上的连接点（通常 1-2 个）
  roomId:   string          // 'wet_well' | 'pump_room'
  canvasX:  number
  canvasY:  number
  isSpare:  boolean
}

// 线（无向）
interface Pipe {
  id:       string
  node1:    string          // 节点 ID
  node2:    string          // 节点 ID
  roomIds:  string[]        // 线所属的房间；跨房间时记录多个
}
```

## 六、用户可操作项（编辑器）

| 操作 | 说明 |
|------|------|
| 拖拽节点 | 自由移动节点位置；节点所在设备同步移动 |
| ＋ 水泵 / ＋ 止回阀 / ＋ 电动闸阀 / ＋ 电磁流量计 / ＋ 节点 | 新增设备或节点 |
| 删除选中 | 选中节点/设备/连线后点击或按 `Delete` 键，关联连线同步删除 |
| 连线模式 | 点击两节点新建 pipe（无向线） |
| 重置 | 恢复当前 N + N_spare 对应的默认拓扑 |

## 七、AG0-1 解析代理输出（`runAG01`）

```ts
{
  valid:    boolean          // 无 errors 时为 true
  errors:   string[]         // 出水口不可达、无工作泵等致命问题
  warnings: string[]         // 孤立节点等提示（无备用泵不再警告）
  stats: {
    N_working:    number     // 工作泵台数
    N_spare:      number     // 备用泵台数
    N_checkValve: number     // 止回阀数量
    N_gateValve:  number     // 电动闸阀数量
  }
  byRoom: {
    [roomId]: { label, devices: Device[] }
  }
  isolated:           Device[]   // 未接入拓扑的孤立设备
  dischargeReachable: boolean    // 能否从进水到达出水口
  topology:           Topology   // 透传给 AG3-1
}
```

**连通性检查**：从进水节点 BFS，检查所有节点可达性和出水口可达性。
