# AG4-3 建筑平面布局评价

> 模块版本：v1.2

---

## 一、模块目标

接收 AG4-2 生成的多个布局方案，对每个方案进行量化评价，输出包含**综合评分**和**空间有效率**的评价结果，供用户选择最优方案。

---

## 二、在系统中的位置

```
AG4-2（建筑平面布局生成，见 AG4建筑平面布局3_泵站业务.md）
  │  输出：LayoutVariant[]（含 placements, violations, adjacency）
  ▼
AG4-3（建筑平面布局评价）← 本模块
  │  输出：EvaluatedVariant[]（含 score, spaceEfficiency, breakdown）
  ▼
UI 展示（方案对比与选择）
```

---

## 三、评价指标

### 3.1 综合评分（score）

| 项目 | 分值 | 说明 |
|------|------|------|
| 基础分 | +10000 | 所有方案起始分数 |
| 占地面积 | −8 pt/m² | 包围盒面积越小得分越高，鼓励紧凑布局 |
| 必须临近满足 | +40/对 | 每满足一对 MUST 邻接关系 |
| 建议临近满足 | +15/对 | 每满足一对 SHOULD 邻接关系 |
| 变压器外墙 | +20/台 | 每台变压器接触东/西外墙 |
| 变压器同侧 | +20 | 两台变压器布置于同侧外墙 |
| 风机房近送货口 | 最高 +30 | 距离越近得分越高（线性衰减） |
| **空间有效率** | 最高 +50 | 见 §3.2，线性映射 |
| **便捷性** | 最高 +30 | 见 §3.3，从入口至各功能房间的步行路径长度 |
| 可达性违反 | −100/间 | 房间开门方向不符合要求，见 §3.4 |
| 房间缺失 | −200/间 | 应出现在该楼层的房间未放置，见 §3.5 |
| 房间长宽比 | −200/间 | 房间长宽比超过 4:1（走廊除外），见 §3.6 |
| MUST 约束违反 | −50/项 | 每项 MUST 邻接关系未满足 |

> **注意**：可达性违反、房间缺失、房间长宽比三项惩罚彼此独立，均以独立字段记录于 `scoreBreakdown`，不混入 `violations`（`violations` 专指 AG4-2 生成阶段的 MUST 邻接约束）。

> **占地面积差异化要求**：不同方案的占地面积（footprint）应有实质性差异。生成侧需确保建筑尺寸/形状在允许范围内产生变化，避免所有方案采用相同的 L × W。

> **方案类型覆盖要求**：输出方案集应包含以下多种类型，不应仅以最小占地为唯一优化目标：
> - 最小整体占地方案
> - 维修区域（`repair_zone`）面积较大的方案（面积 ≥ 120 m² 且尽量大）
> - 停车区域（`parking`）面积较大的方案（面积 ≥ 150 m² 且尽量大）

---

### 3.2 空间有效率（space_efficiency）

```
space_efficiency = Σ(功能房间面积) / (buildingW × buildingD)
```

**功能房间**：所有已放置房间，**除外**：
- 走廊 `corridor_l1`（交通空间）
- 吊装口 `dock1`、`dock2`（楼板开口，非封闭房间）

**含义**：走廊或空白区域越多，有效率越低。

**评分映射（线性）**：

| 空间有效率 | 附加分 |
|-----------|--------|
| ≥ 0.90 | +50 |
| 0.80 | +33 |
| 0.70 | +17 |
| ≤ 0.60 | 0 |

公式：`efficiencyScore = clamp((efficiency - 0.60) / 0.30, 0, 1) × 50`

---

### 3.3 便捷性评分（convenience）

> **命名说明**：本节对应表 3.1 中的「便捷性」指标（最高 +30）。代码实现中该字段历史上命名为 `accessibilityScore` / `breakdown.accessibility`，含义为「入口至各功能房间的步行路径便捷程度」，与本节定义一致。请勿与 §3.4「可达性违反」（房间开门方向约束）混淆。

**定义**：从建筑主入口（地面层主门）至各主要功能房间的步行路径长度（格数，沿走廊/大厅可达路径），取加权平均。

**计算方法**：
1. 以主入口格子为起点，对各功能房间门口格子进行 BFS/最短路径搜索（经走廊/维修大厅）；
2. 取所有目标房间路径长度的加权平均值（权重可按使用频率均等分配）；
3. 路径长度以 100 mm 网格格数计。

> 当前实现以各功能房间质心到名义主入口（地面层南墙中点）的欧氏距离作为路径代理，待门位置数据完善后升级为完整 BFS。

**目标房间**（地面层主要功能房间）：`trafo1`、`trafo2`、`meter_main`、`meter_sub`、`fire_equip`、`parking`、`repair_zone`

**评分映射（线性）**：

| 加权平均路径长度 | 附加分 |
|-------------|--------|
| ≤ 50 格（≤ 5m） | +30 |
| 100 格（10m）   | +15 |
| ≥ 150 格（≥ 15m）| 0 |

公式：`accessibilityScore = clamp((150 - avgPath) / 100, 0, 1) × 30`

---

### 3.4 可达性约束（door access）

**定义**：建筑内每个房间必须通过合规的门与相邻通道连通。

**规则**：

| 楼层 | 房间 | 合规开门方向 |
|------|------|------------|
| 地面层 | trafo1、trafo2、meter_main、meter_sub、fire_equip | 外墙（北/东/西，南墙禁开门） |
| 地面层 | parking、repair_zone | 彼此连通，或向外墙开门 |
| 一层 | fan_room、clean_pump、rainwater、lv_control | 必须向走廊（`corridor_l1`）开门 |

**判定方式**（代理检查）：
- 地面层房间：接触北/东/西外墙视为合规（调用 `touchesExteriorNonSouth()`）；`parking` 与 `repair_zone` 相邻即视为互通合规。
- 一层房间：与 `corridor_l1` 相邻（调用 `adjacent()`）视为合规。

**扣分规则**：每发现一个违规房间扣 100 分。

```
doorAccessPenalty = −100 × violatingRoomCount
```

**代码字段**：`breakdown.doorAccess`（≤ 0）

---

### 3.5 房间完整性惩罚（missing rooms）

**定义**：最终布局必须包含所有规定房间。若某楼层缺少应当出现的房间，每缺失一个扣 200 分。

**地面层必须包含**：

| 房间 ID | 说明 |
|---------|------|
| `trafo1` | 中电变压器房 1 |
| `trafo2` | 中电变压器房 2 |
| `meter_main` | 总水表房 |
| `meter_sub` | 水表房 |
| `fire_equip` | 消防设备房 |
| `parking` | 停车区域 |
| `repair_zone` | 水泵维修区域 |
| `dock1` | 送货口 1（地坑） |

**一层必须包含**：

| 房间 ID | 说明 |
|---------|------|
| `fan_room` | 风机房 |
| `clean_pump` | 清洁泵房及水箱房 |
| `rainwater` | 雨水回收及灌溉设备房 |
| `lv_control` | 低压配电及 PLC 控制室 |
| `dock2` | 送货口 2（吊装口） |
| `corridor_l1` | 一层主走廊 |

**扣分规则**：

```
missingRoomsPenalty = −200 × missingCount
```

**代码字段**：`breakdown.missingRooms`（≤ 0）

---

### 3.6 房间长宽比惩罚（aspect ratio）

**定义**：房间形状应保持合理比例，避免过于狭长。除走廊（`corridor_l1`）外，所有房间的长宽比不得超过 4:1。

**计算**：
```
ratio = max(w, d) / min(w, d)
```
若 `ratio > 4`，该房间视为违规。走廊（`corridor_l1`）豁免此项检查。

**扣分规则**：每发现一个违规房间扣 200 分。

```
aspectRatioPenalty = −200 × violatingRoomCount
```

**代码字段**：`breakdown.aspectRatio`（≤ 0）

---

### 3.7 方案相似度惩罚（diversity penalty）

> **状态：待实现**，在方案集具备足够多样性后启用。

**目的**：避免输出多个高度相似或重复的方案，强制方案集合保持多样性。

**机制**：
- 计算任意两方案间的房间位置相似度（基于各房间中心坐标的归一化距离）；
- 若方案 A 与方案 B 相似度超过阈值（建议初始值 0.85），对得分较低的方案额外扣分（建议 −200 分）；
- 扣分仅在排序时生效，不修改 `score` 字段本身，输出为独立字段 `diversityPenalty`。

## 五、输出参数

```ts
interface EvaluatedVariant extends LayoutVariant {
  score: number              // 综合评分（含所有惩罚和奖励）
  spaceEfficiency: number    // 0~1，功能房间面积占比
  efficiencyScore: number    // 空间有效率对应的附加评分（0~+50）
  accessibilityScore: number // 便捷性附加评分（0~+30）；字段名历史沿用 accessibilityScore，含义见 §3.3
  diversityPenalty: number   // 相似度惩罚（≤0，待实现阶段默认为 0）
  variantType: 'compact' | 'large-repair' | 'large-parking' | 'standard'
               // 方案类型标签（compact=最小占地，large-repair=维修区域大，
               //              large-parking=停车区大，standard=标准方案）
  scoreBreakdown: {
    base: number             // 基础分 +10000
    footprint: number        // 占地面积惩罚（≤0）
    adjacency: number        // 邻接关系奖励（≥0）
    corridor: number         // 走廊邻接奖励（≥0）
    trafo: number            // 变压器外墙/同侧奖励（≥0）
    fanRoom: number          // 风机房近送货口奖励（≥0）
    efficiency: number       // 空间有效率奖励（0~+50）
    accessibility: number    // 便捷性奖励（0~+30）；字段名历史沿用，含义见 §3.3
    violations: number       // MUST 邻接约束违反惩罚（≤0，−50/项）
    doorAccess: number       // 可达性违反惩罚（≤0，−100/间），见 §3.4
    missingRooms: number     // 房间缺失惩罚（≤0，−200/间），见 §3.5
    aspectRatio: number      // 房间长宽比惩罚（≤0，−200/间），见 §3.6
  }
}
```

方案按 `score + diversityPenalty` 降序排列，最优方案排第一。每种 `variantType` 至少保留一个方案展示。

## 六、排除空间说明

以下空间不计入空间有效率分子（功能面积）：

| 空间 ID | 原因 |
|---------|------|
| `corridor_l1` | 走廊为交通空间，非功能房间 |
| `dock1` | 楼板吊装口，无围合面积 |
| `dock2` | 楼板吊装口，无围合面积 |

所有其他已放置房间均计入功能面积。
