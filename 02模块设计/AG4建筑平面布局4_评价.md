# AG4-3 建筑平面布局评价

> 模块版本：v1.1

---

## 一、模块目标

接收 AG4-2 生成的多个布局方案，对每个方案进行量化评价，输出包含**综合评分**和**空间有效率**的评价结果，供用户选择最优方案。

---

## 二、在系统中的位置

```
AG4-2（建筑平面布局生成）
  │  输出：LayoutVariant[]（含 placements, violations, adjacency）
  ▼
AG4-3（建筑平面布局评价）
  │  输出：EvaluatedVariant[]（含 score, spaceEfficiency, breakdown）
  ▼
UI 展示（方案对比与选择）
```

---

## 三、评价指标

### 3.1 综合评分（score）

| 项目 | 分值 | 说明 |
|------|------|------|
| 基础分 | 10000 | 所有方案起始分数 |
| 占地面积 | −8 pt/m² | 包围盒面积越小得分越高，鼓励紧凑布局 |
| 必须临近满足 | +40/对 | 每满足一对 MUST 邻接关系 |
| 建议临近满足 | +15/对 | 每满足一对 SHOULD 邻接关系 |
| 走廊交通完整 | +20 | 走廊同时与 ≥2 个目标房间临近 |
| 变压器外墙 | +20/台 | 每台变压器接触东/西外墙 |
| 变压器同侧 | +20 | 两台变压器布置于同侧 |
| 风机房近送货口 | 最高 +30 | 距离越近得分越高（线性衰减） |
| **空间有效率** | 最高 +50 | 见 3.2，线性映射 |
| **通达性** | 最高 +30 | 见 3.3，从入口至各功能房间的步行路径长度 |
| 约束违反 | −50/项 | 每项 MUST 约束未满足 |

> **占地面积差异化要求**：不同方案的占地面积（footprint）应有实质性差异。生成侧需确保建筑尺寸/形状在允许范围内产生变化，避免所有方案采用相同的 L × W。

> **方案类型覆盖要求**：输出方案集应包含以下多种类型，不应仅以最小占地为唯一优化目标：
> - 最小整体占地方案
> - 维修区域（`repair_zone`）面积较大的方案（面积 ≥ 120 m² 且尽量大）
> - 停车区域（`parking`）面积较大的方案（面积 ≥ 150 m² 且尽量大）

### 3.2 空间有效率（space_efficiency）

```
space_efficiency = Σ(功能房间面积) / (buildingW × buildingD)
```

**功能房间**：所有已放置房间，**除外**：
- 走廊 `corridor_l1`（交通空间）
- 吊装口 `dock1`、`dock2`（楼板开口，非封闭房间）

**含义**：走廊或空白区域越多，有效率越低。

**评分映射（线性）**：

| 空间有效率 | 附加分 |
|-----------|--------|
| ≥ 0.90 | +50 |
| 0.80 | +33 |
| 0.70 | +17 |
| ≤ 0.60 | 0 |

公式：`efficiencyScore = clamp((efficiency - 0.60) / 0.30, 0, 1) × 50`

### 3.3 通达性评分（accessibility）

**定义**：从建筑主入口（地面层主门）至各主要功能房间的步行路径长度（格数，沿走廊/大厅可达路径），取加权平均。

**计算方法**：
1. 以主入口格子为起点，对各功能房间门口格子进行 BFS/最短路径搜索（经走廊/维修大厅）；
2. 取所有目标房间路径长度的加权平均值（权重可按使用频率均等分配）；
3. 路径长度以 100 mm 网格格数计。

**评分映射（线性）**：

| 加权平均路径长度 | 附加分 |
|-------------|--------|
| ≤ 50 格（≤ 5m） | +30 |
| 100 格（10m）   | +15 |
| ≥ 150 格（≥ 15m）| 0 |

公式：`accessibilityScore = clamp((150 - avgPath) / 100, 0, 1) × 30`

> 路径长度统计时仅考虑地面层主要功能房间（`trafo1/2`、`meter_*`、`fire_equip`、`parking`、`repair_zone`）；一层房间通过楼梯折算路径后统一计算。

### 3.4 方案相似度惩罚（diversity penalty）

> **状态：待实现**，在方案集具备足够多样性后启用。

**目的**：避免输出多个高度相似或重复的方案，强制方案集合保持多样性。

**机制**：
- 计算任意两方案间的房间位置相似度（基于各房间中心坐标的归一化距离）；
- 若方案 A 与方案 B 相似度超过阈值（建议初始值 0.85），对得分较低的方案额外扣分（建议 −200 分）；
- 扣分仅在排序时生效，不修改 `score` 字段本身，输出为独立字段 `diversityPenalty`。

## 五、输出参数

```ts
interface EvaluatedVariant extends LayoutVariant {
  score: number              // 综合评分
  spaceEfficiency: number    // 0~1，功能房间面积占比
  efficiencyScore: number    // 空间有效率对应的附加评分（0~50）
  accessibilityScore: number // 通达性附加评分（0~30）
  diversityPenalty: number   // 相似度惩罚（≤0，待实现阶段默认为 0）
  variantType: 'compact' | 'large-repair' | 'large-parking' | 'standard'
               // 方案类型标签（compact=最小占地，large-repair=维修区域大，
               //              large-parking=停车区大，standard=标准方案）
  scoreBreakdown: {          // 各项得分明细
    base: number
    footprint: number
    adjacency: number
    corridor: number
    trafo: number
    fanRoom: number
    efficiency: number
    accessibility: number
    violations: number
  }
}
```

方案按 `score + diversityPenalty` 降序排列，最优方案排第一。每种 `variantType` 至少保留一个方案展示。

---

## 六、排除空间说明

以下空间不计入空间有效率分子（功能面积）：

| 空间 ID | 原因 |
|---------|------|
| `corridor_l1` | 走廊为交通空间，非功能房间 |
| `dock1` | 楼板吊装口，无围合面积 |
| `dock2` | 楼板吊装口，无围合面积 |

所有其他已放置房间均计入功能面积。
