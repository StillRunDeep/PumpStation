# AG4-3 建筑平面布局评价

> 模块版本：v1.6（统一评分与动态多样性惩罚）

---

## 一、模块目标

接收 AG4-2 生成的多个布局方案，对每个方案进行量化评价。评价体系不仅关注物理约束（MUST 邻接），还引入了空间有效率、便捷性、形状利用率等多个维度的量化指标，最终输出包含**综合评分**、**多样性惩罚**及**详细得分明细**的评价结果，供用户选择最优方案。

---

## 二、在系统中的位置

```
AG4-2（建筑平面布局生成，见 AG4建筑平面布局3_泵站业务.md）
  │  输出：LayoutVariant[]（含 placements, violations, adjacency）
  ▼
AG4-3（建筑平面布局评价）← 本模块 (src/layout/scorer.js)
  │  输出：EvaluatedVariant[]（含 score, breakdown, variantType）
  ▼
UI 展示（src/ui/layout-panel.js，展示排名与得分明细）
```

---

## 三、排序与合并规则

本系统采用**全球统一评分排名**机制，确保用户看到的始终是当前最优的 9 个方案。

1.  **统一池化**：每次新生成的 9 个方案与现有的 9 个方案合并，形成一个包含 18 个候选者的池子。
2.  **绝对分值导向**：废除之前的“可行性分桶”逻辑（不再优先保排 violations=0 的方案）。所有方案均按最终综合得分（Base 10000 + 奖励 - 惩罚）进行降序排列。
3.  **多样性洗牌**：
    *   在选出最终 Top 9 之前，系统会对比池中排名靠前方案的几何相似度。
    *   如果一个新方案与排名更高（分更高）的方案过于相似（重心平均距离小于`diversityThreshold`），则该方案将被扣除`diversityPenalty`。
    *   扣分后，方案会重新参与排名。这确保了榜单前列不会出现重复或高度相似的布局，强制推动算法探索更多可能性。

## 四、评价指标体系（Breakdown 详解）

评分采用**基础分 + 奖励分 - 惩罚分**的累计方式。

### 第一梯队：硬性功能红线（Hard Redlines）
此类指标若未满足，将产生巨大的分值塌陷（通常 ≥ 500pt），使方案跌出榜单。

| 指标字段 | 建议分值 | 目的与逻辑 |
| :--- | :--- | :--- |
| **缺失房间 (`missingRooms`)** | **−500 pt / 间** | 最严重错误。设备无处安放，方案直接视为无效。 |
| **可达性违规 (`doorAccess`)** | **−500 pt / 间** | 物理隔绝。如服务间不靠外墙、设备间不靠走廊。 |
| **MUST 约束违反 (`violations`)** | **−500 pt / 项** | 违反明确的业务红线（如变压器室未相邻）。 |
| **多样性惩罚 (`diversity`)** | **-200 pt (默认)** | 用户可调。防止榜单被相似方案霸占。 |

---

### 第二梯队：空间质量生命线（Spatial Quality）

| 指标字段 | 建议分值 | 目的与逻辑 |
| :--- | :--- | :--- |
| **形状与比例 (`aspectRatio`)** | **−500 pt / 单位违规** | 抑制畸形（长宽比>4、利用率低、顶点数>6）。 |
| **空间有效率 (`efficiency`)** | **最高 +50 pt** | 衡量建筑紧凑度（功能面积/总面积）。 |
| **走廊完整性 (`corridor`)** | **最高 +20 pt** | 鼓励形成清晰的交通枢纽。 |

---

### 第三梯队：优化项（Optimization）

| 指标字段 | 建议分值 | 目的与逻辑 |
| :--- | :--- | :--- |
| **MUST 临近满足 (`adjacency`)** | **+40 pt / 对** | 满足 MUST 关系提供基础奖励。 |
| **SHOULD 临近满足 (`adjacency`)** | **+15 pt / 对** | 建议性的功能靠近。 |
| **便捷性 (`accessibility`)** | **最高 +30 pt** | 缩减入口至各房间的平均步行距离。 |
| **变压器布置 (`trafo`)** | **最高 +40 pt** | 奖励变压器靠外墙或同侧布置。 |
| **风机房距离 (`fanRoom`)** | **最高 +30 pt** | 奖励风机房靠近一层吊装口。 |
| **生长顺利度 (`growthSuccess`)** | **最高 +100 pt** | 奖励生长充分（面积达标）的布局。 |

## 五、输出结构定义

```ts
interface EvaluatedVariant {
  score: number              // 最终总得分
  spaceEfficiency: number    // 空间有效率原始值 (0~1)
  efficiencyScore: number    // 空间有效率对应的奖励分
  accessibilityScore: number // 便捷性对应的奖励分
  aspectRatio: number        // 方案中所有房间的最大长宽比
  diversityPenalty: number   // 相似度惩罚（由 mergeVariants 在排名后统一应用）
  
  breakdown: {
    base: number             // 基础分 10000
    diversityPenalty: number // 多样性扣分
    // ... 其他指标分值 (详见代码)
  }
}
```
