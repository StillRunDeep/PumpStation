# 最终方案：线段优先的智能填充 (Boundary-First Filling)

## 1. 最终问题定义

经过多轮深入探讨，我们确定了问题的最终症结：为了获得平滑、美观、专业的建筑平面图，必须在**数据生成阶段**就主动创造出简洁的房间边界，而不是在渲染时被动地去美化一个已经很复杂的形状。

核心挑战在于，如何在填充“约束生长法”留下的空白区域时，做出最有利于**全局边界简化**的决策。

## 2. 核心方案：线段优先填充法

该方案将完全重写现有的`fillGaps`函数。其革命性的改变在于，填充的基本单位不再是**单个格子**，而是**连续的水平或垂直的空白格子线段**。

算法通过一个迭代过程，在每一步都选择并填充当前最优的线段，直至所有空白区域被填满。

### 2.1. 算法流程

在每一次迭代中：

1.  **识别所有可填充线段**：找出当前所有最长的、连续的水平和垂直空白线段。
2.  **评估与评分**：对每一条线段，应用下述的**两阶段混合评分法**，找到最佳的填充方案（即决定把线段分配给哪个邻居房间）。
3.  **选择最优并填充**：选择全局得分最高的线段-房间组合，执行填充。
4.  **重复**：回到步骤1，直至没有空白格子。

### 2.2. 两阶段混合评分法 (方案 B+C)

这是整个算法的灵魂，确保每一个填充决策都服务于边界简化的最终目标。

#### 阶段一：优先填充“凹角” (方案 C)

在进行常规评分前，算法会首先扫描并处理掉所有**最高优先级**的填充机会。

-   **识别模式**：寻找那些被**同一个房间**从三个方向（或两个长边和一个短边）包围的空白线段。
-   **操作**：立即将这些线段分配给那个包围它的房间。
-   **目的**：这相当于在“填补房间的凹角”，是确定性最强、收益最高的简化操作，能快速地让房间形状变得“饱满”和“方正”，为后续决策打下良好基础。

#### 阶段二：以“边界简化”为目标的常规评分 (方案 B)

对于所有未被阶段一处理的线段，算法将进行详细的量化评分。

-   **核心指标**：**直接量化一次填充对边界复杂度的改善程度。** 复杂度用“转折点数量”来衡量。
-   **分数计算**：
    1.  对一个候选线段，以及它的一个邻居房间（如 `room_A`）。
    2.  计算如果将线段分配给 `room_A`，会**消除**多少个 `room_A` 的旧转折点。
    3.  计算此举会**创造**多少个 `room_A` 的新转折点。
    4.  `Score = (消除的转折点数) - (创造的转折点数)`
-   **决策**：
    -   算法将对所有可能的线段-房间组合计算这个分数。
    -   最终选择那个能提供**最高正分**（即最大程度净减少转折点）的组合进行填充。
    -   **决胜局 (Tie-Breaker)**：在分数相同的情况下，优先选择**更长**的线段进行填充。

## 3. 渲染策略 (维持不变)

在数据层面通过上述算法得到边界平滑的房间布局后，渲染阶段依然采用**矩形合并 (`decomposeRoomIntoRects`)** 的方案。简洁的房间形状数据可以被该算法高效地分解为一组干净的大矩形，从而实现最佳的视觉渲染效果。

## 4. 方案优势

-   **目标驱动**：算法的每一个决策都由“减少转折点”这一最终目标所驱动。
-   **宏观视角**：以“线段”而非“格子”作为操作单元，从宏观上保证了边界的连续性与平直性。
-   **智能与前瞻性**：通过“预计算”填充效果，算法能做出具有前瞻性的、能带来全局最优的决策。
-   **结果优越**：能够产生远比简单填充更自然、更合理、更符合专业设计直觉的平滑边界。
